# PDF To Markdown

`mdpdf reverse` extracts text from a PDF and reconstructs readable Markdown.

This mode is intentionally described as best effort. PDF files are presentation
documents: they usually store positioned text, fonts, and drawing commands, not
the original Markdown, HTML, or word processor structure.

## Usage

```bash
mdpdf reverse input.pdf
```

This writes `input.md` next to the source file.

Set an explicit output file:

```bash
mdpdf reverse input.pdf -o extracted.md
```

Preserve page boundaries:

```bash
mdpdf reverse input.pdf -preserve-pages -o extracted.md
```

Page boundaries are written as HTML comments:

```markdown
<!-- page 2 -->
```

## What It Can Reconstruct

The reverse mode uses PDF text positions, font names, and font sizes to infer:

- Headings from large bold text.
- Paragraphs from line spacing.
- Unordered lists from bullet markers.
- Ordered lists from numbered markers.
- Code blocks from monospace fonts.
- Blockquotes from italic text.
- Simple tables from repeated aligned text chunks.

## Known Limits

Some PDFs do not contain extractable text. Scanned documents usually need OCR
before this tool can help.

Even with extractable text, the output may need manual cleanup:

- Multi-column layouts can produce unexpected reading order.
- Complex tables can flatten into paragraphs or simple tables.
- Headers, footers, page numbers, and footnotes may be mixed into body text.
- Images are not exported as files.
- Links may appear as plain text.
- Original Markdown formatting is not recoverable with certainty.

## Recommended Workflow

1. Run `mdpdf reverse input.pdf -o draft.md`.
2. Read the Markdown once for ordering and heading levels.
3. Clean up tables, code indentation, and any page artifacts.
4. Convert the cleaned Markdown back to PDF if needed.

For PDFs generated by `mdpdf`, reverse mode should usually produce clean and
readable Markdown because the writer uses simple text placement and built-in
fonts.
