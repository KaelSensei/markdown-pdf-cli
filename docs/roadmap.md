# Roadmap

This file tracks practical next steps for `mdpdf`. It is intentionally focused
on concrete product hardening work rather than broad ideas.

## Next Release

- Publish `v0.2.0`.
- Verify that the release workflow produces Linux, macOS, and Windows binaries.
- Download and smoke-test each generated binary.
- Confirm that the GitHub Release notes are clear enough for first-time users.

## Real-World PDF Testing

Test reverse conversion against PDFs that were not generated by `mdpdf`:

- A simple text-heavy PDF.
- A PDF with basic tables.
- A PDF with multi-column layout.
- A PDF with headers, footers, and page numbers.
- A scanned PDF to verify and document the no-OCR behavior.

Record any representative failure cases as tests or fixtures when licensing
allows it.

## PDF To Markdown Improvements

- Improve code block indentation preservation.
- Improve detection of complex tables.
- Add optional header, footer, and page number filtering.
- Improve reading order for multi-column pages.
- Add configurable extraction modes such as `plain`, `structured`, and
  `preserve-pages`.
- Investigate optional OCR integration for scanned PDFs.

## Markdown To PDF Improvements

- Add embedded TrueType font support for broader Unicode coverage.
- Improve code block indentation and spacing.
- Add optional syntax highlighting for fenced code blocks.
- Add optional table of contents generation.
- Improve table layout for wide and wrapped content.
- Add better handling for nested lists.

## Repository Quality

- Add `CONTRIBUTING.md`.
- Add `CHANGELOG.md`.
- Add issue templates.
- Add a pull request template.
- Consider `golangci-lint` once the project has enough rules to justify it.
- Keep examples small but representative.

## Future Product Directions

- Batch conversion for whole directories.
- A web UI built on top of the existing conversion core.
- A desktop wrapper if local non-terminal usage becomes important.
- Optional HTML/CSS rendering backend for users who need richer layout.
